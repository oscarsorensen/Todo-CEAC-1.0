Enunciado paso a paso
Introducción y contextualización Jose Vicente, un apasionado del gimnasio y el cocina, ha estado trabajando en su proyecto personal de programación. Hoy, ha llegado a un punto donde necesita almacenar información entre diferentes páginas web para mantener la experiencia coherente.

Desarrollo técnico correcto y preciso Jose Vicente ha creado dos archivos PHP, origen.php y destino.php, para probar cómo pasar variables entre páginas. Sin embargo, se ha dado cuenta de que las variables no persisten entre ellas.

Aplicación práctica con ejemplo claro Jose Vicente necesita modificar su código para que la variable $nombre almacenada en origen.php sea accesible en destino.php. Para lograr esto, debe utilizar sesiones en PHP.

Cierre/Conclusión enlazando con la unidad Una vez que Jose Vicente tenga su variable persistiendo entre páginas, podrá aplicar estos conocimientos en su proyecto final de programación, asegurándose de que el usuario pueda interactuar coherentemente a lo largo de la aplicación.

Restricciones
No se permite usar variables globales.
Solo puede utilizar funciones y estructuras vistas hasta ahora en clase.
No se pueden utilizar librerías externas ni estructuras no vistas.
Criterios de evaluación
Introducción y contextualización (25%): El estudiante debe mostrar una comprensión adecuada del contexto y la necesidad de persistir variables entre páginas.
Desarrollo técnico correcto y preciso (25%): El estudiante debe utilizar sesiones en PHP para almacenar y recuperar variables entre diferentes páginas.
Aplicación práctica con ejemplo claro (25%): El estudiante debe presentar un código funcional que demuestre la persistencia de variables utilizando sesiones.
Cierre/Conclusión enlazando con la unidad (25%): El estudiante debe relacionar el ejercicio con su proyecto final y explicar cómo aplicará estos conocimientos en el futuro.
